# Contexto do Projeto — Engineering Operating System (EOS)

## 1. O Problema
Equipes de tecnologia frequentemente enfrentam inconsistência na qualidade de código, tomadas de decisão arquiteturais ad-hoc sem embasamento em trade-offs, refatorações inseguras e falta de governança técnica unificada ao gerenciar múltiplos projetos de software de escalas diferentes.

## 2. A Solução (O que é o EOS)
O EOS é um "sistema operacional de engenharia" que atua como um framework metodológico para organizar princípios, processos, padrões, protocolos, modelos de decisão e templates. Ele visa padronizar a governança de software sem impor restrições de tecnologia ou stack, focando no refinamento do pensamento técnico e na qualidade das decisões arquiteturais.

## 3. Restrições e Limites do Projeto
* **Agnóstico de Tecnologia**: O EOS não deve prescrever bibliotecas, frameworks específicos de linguagem de programação ou infraestruturas particulares em seu core.
* **Fácil Adoção**: A aplicação do EOS em um projeto deve requerer apenas a criação de uma pasta `.eos/` com arquivos markdown leves.
* **Evolução como Software**: Alterações no EOS devem passar por revisão, versionamento semântico e justificativas técnicas claras.
